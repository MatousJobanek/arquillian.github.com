Arquillian Cube Extension 1.0.0.Alpha18 Released

Posted by Alex Soto on Feb 21, 2017
Feb 21
| Comments

The Arquillian team is proud to announce the 1.0.0.Alpha18 release of the Arquillian Cube Extension component!

New version of Arquillian Cube 1.0.0.Alpha18 has been released!

What’s new

Arquillian Cube BOM

Arquillian Cube BOM has been released so you can unify naming and versions of Arquillian Cube.

<properties>
    <version.arquillian_cube>${latest_released_version}</version.arquillian_cube>
</properties>

<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>org.arquillian.cube</groupId>
            <artifactId>arquillian-cube-bom</artifactId>
            <version>${version.arquillian_cube}</version>
            <scope>import</scope>
            <type>pom</type>
        </dependency>
    </dependencies>
</dependencyManagement>

Then you can include the individual modules such as:

<dependency>
    <groupId>org.arquillian.cube</groupId>
    <artifactId>arquillian-cube-docker</artifactId>
    <scope>test</scope>
    <type>pom</type>
</dependency>

Add AssertJ matcher for current running processes

You can assert on current running processes on given container by doing: assertThat(docker).container("mycontainer").hasProcessRunning("name of process")

Kudos to Ioannis Canellos for internal enhancements on Kubernetes and OpenShift integration.

What is Arquillian Cube Extension?

With Arquillian Cube you can control the lifecycle of Docker images as part of the test lifecyle, either automatically or manually. This gives you the chance to scale up from a integration/functional test level all the way up to the system test level.

ShrinkWrap Resolver 3.0.0-alpha-4 Released

Posted by Matous Jobanek on Feb 20, 2017
Feb 20
| Comments

The Arquillian team is proud to announce the 3.0.0-alpha-4 release of the ShrinkWrap Resolver component!

In this release, we optimized a storage of downloaded Maven binaries and fixed scope of the JUnit dependency.
This will make ShrinkWrap Resolver more stable and ensure that the test dependencies won’t be transitively fetched in your project.

What is ShrinkWrap Resolver?

The ShrinkWrap Resolvers project provides a Java API to obtain artifacts from a repository system. This is handy to include third party libraries available in any Maven repository in your test archive. ShrinkWrap Resolvers additionally allow you to reuse all the configuration you've already specified in the Maven build file, making packaging of an application archive much easier job.

Arquillian Universe 1.1.12.0 Released

Posted by Alex Soto on Feb 15, 2017
Feb 15
| Comments

The Arquillian team is proud to announce the 1.1.12.0 release of the Arquillian Universe component!

New version of Arquillian Universe 1.1.12.0 has been released!

What’s new

Version convention

A new version strategy has been defined for Arquillian Universe. Instead of having alpha versions incrementing the number of alpha, new strategy will have following scheme:

With Arquillian Core 1.1.12 released and no new extensions included in universe yet, the version for universe would be 1.1.12.0. When, let’s say we have new Cube-Q version but the core is still untouched we bump last digit and release 1.1.12.1.

In essence the first three parts of the version corresponds to Arquillian Core that is compatible with current universe. The last section is an increment of universe still being compatible with core but with new/updated extensions.

Arquillian Cube Kubernetes

With this release we added Arquillian Cube Kubernetes support.

<dependency>
  <groupId>org.arquillian.universe</groupId>
  <artifactId>arquillian-cube-kubernetes</artifactId>
</dependency>

What is Arquillian Universe?

The Arquillian Universe BOM is a 'Bill of Material' Maven POM file collection that make up a module stack that stretch across the whole Arquillian Universe.
